# Issues & Projects

## Viewing issues

The **Issues** tab shows all issues across the team's visible projects. You can switch between two layouts:

- **List view** — Issues grouped by status with expandable/collapsible sections
- **Board view** — Kanban-style columns, great for seeing workflow at a glance

Use the toggle in the top-right of the toolbar to switch views.

## Grouping

Issues can be grouped in two ways:

- **By Status** — Groups like Backlog, In Progress, Done (default)
- **By Label** — Groups by the labels assigned to each issue

Toggle between grouping modes using the icons next to the view toggle. Issues with multiple labels will appear in each relevant group.

## Filtering

Click **Filter** in the toolbar to reveal filter options:

- **Status** — Show only specific statuses
- **Priority** — Filter by urgency level
- **Project** — Narrow down to a specific project
- **Label** — Show only issues with certain labels
- **Cycle** — Filter by sprint/cycle

Active filters show as highlighted badges. Click **Clear** to reset all filters.

## Searching

Use the search box to find issues by title or identifier (e.g., "PRJ-123").

## Issue details

Click any issue to open the detail panel on the right side. This shows:

- Full description (with Markdown formatting)
- Status, priority, and labels
- Due date and project assignment
- Activity timeline with status changes and comments

## Projects tab

The **Projects** tab lists all visible projects with:

- Progress bar showing completion percentage
- Current status badge
- Target date (if set)

![Projects tab showing project list with progress](/docs/projects-tab.png)
*Projects listed with status badges, progress bars, and target dates.*

Click a project to see its dedicated page with issues scoped to that project.
